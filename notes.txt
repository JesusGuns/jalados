El único comportamiento que debes tener en cuenta es que, en algunos navegadores, al usar window.location.search tras un rewrite, la barra de direcciones del usuario podría pasar de mostrar /xv-sofia-castro/xfsdefff a mostrar /xv-sofia-castro/index.html?token=xfsdefff justo después de que cargue la página.

Si notas que eso te pasa y quieres que la barra de direcciones se quede 100% limpia y 
fija en [www.moments.com/xv-sofia-castro/xfsdefff](https://www.moments.com/xv-sofia-castro/xfsdefff) sin que el invitado vea jamás el signo de interrogación, puedes agregar esta pequeña línea de código justo después de obtener tu token:

const params = new URLSearchParams(window.location.search);
const token = params.get("token") || "";

if (token) {
    // ESTA LÍNEA OCULTA EL "?token=..." DE LA BARRA DE DIRECCIONES SIN BORRAR LA VARIABLE
    window.history.replaceState({}, document.title, window.location.pathname.replace('/index.html', ''));
    
    // Aquí sigues con tu lógica normal hacia Google Sheets
    obtenerInfoInvitado(token);
